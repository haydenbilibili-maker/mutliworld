## 二、关键差距（按优先级）

### 差距 1 —— 实时航班/货运密度严重不足（#3，最高优先级）

**现象**：实时航班基本不展示；远未达到 WM 截图 4 的航迹密度。

**根因（已定位）**：航班数据走 OpenSky `states/all`，仅支持匿名 / 已弃用的 Basic 认证。
OpenSky 已转向 OAuth2 客户端凭证，匿名访问被严格限流（常 429/空响应），导致航班稀疏或为空。

**修复方案**：
1. **换免鉴权社区 ADS-B 源**（推荐）：`adsb.lol` / `adsb.fi` / `adsb.one`，全球覆盖、无需密钥、支持按区域拉取，密度可达 WM 水平。
2. 或 **接 OpenSky OAuth2**：注册应用获取 client credentials，免费额度显著高于匿名。
3. **货运/海运实时**：免费全球 AIS 受限（MarineTraffic/VesselFinder 多为付费）。可用 `aisstream.io`（免费 WebSocket，需 key）或保留"航运通道密度模拟 + 真实港口/要道节点"的混合方案，并明确标注。
4. **密度优化**：按视野 bbox 动态拉取、视口外不渲染、聚合/抽稀，刷新 ~30–60s。

### 差距 2 —— 图标体系：emoji → 渲染型动画标记（#2，高优先级）

**现象**：当前标记大量使用 emoji（🚫⚡🛰️…）。WM 使用的是按量级缩放的**纯色圆点 + 光晕 + 计数聚合 + 航向箭头**，更统一、更"活"。

**重构方向**：
- **量级圆点**：circle 层按 impact/magnitude 缩放半径 + 颜色分级（红橙黄）+ 外发光（circle-blur 光晕），替代 emoji 主视觉。
- **聚合计数**：高密度区域聚合为带数字的簇（WM 截图中的「29」「166」圆），点击展开。
- **航向箭头**：航班/船舶用方向性箭头图标（随 heading 旋转），而非静态 emoji。
- **动效**：新事件脉冲扩散、选中高亮呼吸、航迹流动；保留少量 emoji 仅作语义点缀。
- **交互**：统一 hover 浮窗（WM 的极简黑底卡：标题+关键参数，如「AFR902 · 35,002ft · 463kts · 165°」「Monet · Undersea Cable」）。

### 差距 3 —— 实时"活地图"观感

- WM 的航迹连线、事件脉冲、底图微动让地图"活"起来；OmniLens 偏静态。
- 优化：航班尾迹（淡出折线）、卫星轨道环动效（已有部分）、事件入场动画、跑马灯与地图联动高亮。

### 差距 4 —— 性能与密度承载

- 大量点/线渲染需保证流畅：circle/symbol 批量层（已用）、bbox 裁剪、聚合、按 zoom 抽稀、避免 DOM marker 滥用（仅空间站等少量用 DOM）。

### 差距 5 —— 细节打磨

- 2D/3D 切换的丝滑过渡；图例与图层联动；移动端适配；时间轴回放（WM 顶部时间）；分享/截图。

## 四、差异化护城河（中国版本的"超越"点）

OmniLens 不必只做 WM 复刻，已有的差异化优势可放大为护城河：
1. **三层级联**（天—地—海）与垂直剖面：WM 无洋底/宇宙纵深。
2. **能源经济实数据**：FRED/EIA/WorldBank/Stooq，为投资/政策预估打底。
3. **中国视角**：国保单位、北斗/中国深空网、国情外链、青金视觉、中立并陈各国航天。
4. **真实数据红线**：已完成"种子/示例"占位正名为真实公开态势数据。
5. **多天体探索**：月球/火星人类探索痕迹，WM 完全没有。

> 结论：核心数据底座与图层广度已具备对标基础；**最短的两块板是「实时航班密度（#3）」与「图标动效体系（#2）」**。补齐这两项后，OmniLens 在观感上即可与 WM 平视，并凭三层级联/能源经济/中国视角形成差异化超越。
