## 2. maplibre 4 → 5 升级评估

### 2.1 主要收益
- `projection: { type: 'globe' }` 真 3D 地球（低 zoom 自动球面、高 zoom 平滑过渡到墨卡托）。
- 这是 World Monitor 最直观的招牌视觉，也是宇宙/全球态势最自然的承载。

### 2.2 Breaking changes（v5）需回归的点
1. **最低浏览器/WebGL 要求**提高（WebGL2）——现代浏览器均满足。
2. 少量已废弃 API 移除；本项目用到的 `Map / addSource / addLayer / GeoJSONSource / Popup / NavigationControl / setLayoutProperty / flyTo / ResizeObserver` 均为稳定 API，预计无碍。
3. 自定义 `StyleSpecification`（本项目 MapContainer 手写 style）在 v5 仍兼容；globe 可通过 style 顶层 `projection` 或运行时 `map.setProjection` 开启。
4. 符号/光晕/线图层在球面下渲染正常；但**贴地大范围矩形/全屏栅格**（如 EMODnet 海床瓦片）在球面边缘可能轻微变形——洋底层默认平面，不冲突。
5. `icon-allow-overlap` 等 symbol 行为在球面下表现一致。

### 2.3 风险与回滚
- 风险：major 升级可能引入个别样式/类型细节问题 → 由 Turing 用 Codex 异源复核 + `npx tsc --noEmit` + 手动回归（切层/弹窗/飞行/区域边界）。
- 回滚：`npm i maplibre-gl@^4.7.1` 一键回退；globe 代码因 shim 包裹，回退后自动降级为 no-op，不影响平面功能。

## 3. 已落地的前向兼容实现（本次）

| 文件 | 作用 |
|------|------|
| `src/store/useMapStore.ts` | 新增 `globe: boolean` + `setGlobe()`；空间层（space tier）自动建议开启 |
| `src/components/map/GlobeController.tsx` | 监听 `globe` 开关 / 当前层；通过 `(map as any).setProjection?.(...)` 安全调用——**v4 下为 no-op，v5 下启用球面**；try/catch 包裹 |
| `src/components/ui/TierSwitcher.tsx` | 新增「🌐 3D 地球」开关按钮 |
| `MapContainer` | 渲染 `<GlobeController/>`（MapProvider 内） |

**设计要点**：
- 进入 🛰 宇宙空间层时自动建议球面视角（最贴「上天看地球」叙事）；其余层保持平面，用户可手动切。
- 所有 globe 调用走 `setProjection?.` 可选链 + try/catch，**绝不破坏 v4 运行**。
- `npx tsc --noEmit` 在 v4 下保持 0 错误（用 `as any` shim 规避 v4 缺失类型）。

## 5. 球面专属增强（已前向兼容落地，v5 点亮即生效）

| 文件 | 增强 | v4 行为 |
|------|------|---------|
| `GlobeController.tsx` | 运行时探测 `isGlobeSupported`（仅 v5 有 setProjection）；球面时 `setSky` 大气辉光 + `setPaintProperty` 背景透明（露出星空） | 探测为 false → 全部跳过，平面观感零变化 |
