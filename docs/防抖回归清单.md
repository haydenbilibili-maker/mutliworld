# 防抖回归清单（提交前必跑）

> 目的：防止刷新、视野同步、图层动效等再次引入 UI 闪烁/抖动。**每次提交前必须执行。**

## 一键执行

```bash
npm run precommit        # = tsc --noEmit + 防抖静态扫描
# 或单独：
npm run check:antijitter
```

`check:antijitter` 扫描已定位过的抖动反模式；**有 ERROR 必须修复后再提交**，WARN 需人工确认（已知安全可在脚本 `WARN_EXEMPT` 登记）。

## 已知抖动根因与对策（历史）

| 根因 | 对策 |
|---|---|
| 视野同步用 `router.replace`（触发 RSC 软导航整树重渲染，刷新/周期抖动） | 改用 `window.history.replaceState` |
| `ResizeObserver` 对细微尺寸变化也触发 | 加尺寸阈值守卫（<2px 跳过） |
| 宇宙动效 `CosmicGlobeAnimator` 在非宇宙层仍运行 | 仅 `activeTier==='space' && isEarth` 时启用 |
| 地图层逐帧 `setPaintProperty/setData` 持续重绘 | 仅 `idle` 时驱动 + 可见性门控 + 限帧 + 尊重 reduced-motion（见 FlowLayer） |
| 底图 `setStyle` 散落多处 | 仅由 `BasemapController` 统一调用 |
| 选择器返回新对象导致频繁重渲染 | 选择原始值/`useShallow`，原子更新 viewport |

## 联机视觉核验（大版本/动效变更后）

静态扫描之外，涉及地图动效/底图/视野的改动，push 后用浏览器复核：

1. 刷新页面 3 次，观察加载初期与稳定后**无周期性抖动/闪烁**。
2. 切换 地表/洋底/宇宙 与 天体，底图切换后图层正常重建、无空白、无跳动。
3. 开启实时航班/连线流光等动效，平移地图时不卡顿；静止时动效平滑。
4. 打开/关闭面板、生成简报（后台）时，地图视野稳定不跳。
